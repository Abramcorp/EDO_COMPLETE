## Зафиксированные правила работы (CRITICAL)

1. **При редактировании PDF — РЕАЛЬНО удалять символы из content stream** (заменять `<0010>` на `<0003>` пробел в TJ-массивах, удалять Tj-операторы), НЕ закрывать белыми прямоугольниками. Белая заливка — костыль, оставляет невидимые символы и нарушает text-extraction.

2. **Рендер PDF только в максимальном разрешении (600 DPI)**. Сравнение и анализ координат — попиксельно в 600 DPI. JPG для preview можно сохранять с quality=92-95.

3. **Координаты штампа ЭДО — ФИКСИРОВАНЫ** (как в эталоне Романов УСН 2025). НЕ менять. Если штамп заезжает на контент — править структуру шаблона, а не штамп.

4. **Соответствие приказа периоду**:
   - 2023 и ранее → приказ 25.12.2020 № ЕД-7-3/958@ (legacy)
   - 2024 → приказ 02.10.2024 № ЕД-7-3/813@ (template_2024.xlsx, XML 5.08)
   - 2025 → приказ 26.11.2025 № ЕД-7-3/1017@ (template_2025.xlsx, XML 5.09)
   - Тензор-эталон Романова за 2025 использует версию 5.08 (приказ 26.11.2025 ещё не действовал на момент сдачи)

5. **Декларация УСН (новая форма 2024+)**: КБК НЕ указывается — только ОКТМО + суммы. ИП в "Достоверность подтверждаю" НЕ заполняет ФИО — только подпись+дата. Текст ЗАГЛАВНЫМИ. Прочерк "-" в каждом незаполненном знакоместе. Стоимостные показатели — в полных рублях.

6. **Git патчи**: если несколько правок — ОДИН пакетный патч. Разбивать только если есть объективная причина (циклические зависимости, тесты должны пройти между, миграция БД отдельно) — и явно объяснить почему.

7. **КПП на стр.1 reference уже пустой** (нет прочерков). Для ИП КПП всегда пустой — этим полем filler не занимается.

## Зафиксированный технический метод (THE METHOD)

### Координатное преобразование

```python
# fields_2024.json использует reportlab pt (origin bottom-left, Y растёт вверх)
# Эталон Тензора и blank используют виртуальные tm-coords (после cm 0.24...841.92, потом cm 2.93...58.33)

def rl_to_tm(x_rl, y_rl):
    """ReportLab pt → виртуальные tm-coords для записи через Tm matrix."""
    tm_x = (x_rl - 14) / 0.7043
    tm_y = (827.92 - y_rl) / 0.7043 - 2.555  # калибровка
    return tm_x, tm_y
```

Калибровка проверена на 10 полях из fields_2025.json, ΔX < 0.05 pt, ΔY = 0 после поправки −2.555.

### Шрифт `/FX` (CIDFontType2)

Встроен Liberation Sans subset 87 символов с Identity-H encoding.

```python
# Структура шрифта:
# /FX (Type0) → /CIDFontType2 (DescendantFonts) → /FontDescriptor → /FontFile2 (TTF)
# CIDToGIDMap = /Identity → CID == GID
# Получение CID для символа: 
unicode_to_gid = {ord(c): gid for c, gid in subset_data.items()}
def char_to_cid_hex(ch):
    return f'{unicode_to_gid[ord(ch)]:04X}'
```

Готовый встроенный шрифт лежит в `blank_with_FX.pdf` под именем `/FX` на всех 4 страницах.
